# Sales Coach Bot

**An AI-powered Telegram bot that runs a daily micro-training program for your sales team, with a manager dashboard, leaderboards, redeemable rewards, and a weekly performance report.**

> Open-source shell you can fork, brand, and adapt to your company in about an hour.

---

## What it does

Every workday, each rep receives 3-5 short questions on Telegram about your product, sales methodology, or common objections. They answer with free text or voice notes. The bot grades the answer using an LLM against your knowledge base, gives coaching feedback on why the answer was right or wrong, awards points, and tracks the rep's weekly precision. Every Friday, a leaderboard email is sent to managers with a full performance report and week-over-week deltas.

Reps see their rank, their streak, and where they stand vs. teammates — which is what actually drives engagement.

## Features

- **Daily Telegram quizzes** — 3-5 questions per rep per day, timezone-aware per country
- **Hybrid grading** — LLM (GPT-4o-mini or Claude) validates open-ended answers against your knowledge base, with keyword pre-checks for cost control
- **Adaptive probing** — targeted follow-up questions when a rep misses a concept
- **Coaching feedback** — every wrong answer includes a "why it matters" line generated by the LLM
- **Weekly precision tracking** — reps see their accuracy trend on every answered question
- **Manager dashboard** — Flask web app with rankings, cuts by country/team/executive, common errors, and inactivity alerts
- **Points & rewards** — reps earn points, keep streaks, and redeem for prizes you configure
- **Weekly HTML email report** — sent Fridays via Resend (HTTPS) or SMTP, with a competitive leaderboard designed for reps, not just managers
- **Feedback loop** — reps can flag confusing or incorrect questions right from Telegram
- **Multi-country support** — each rep has a base country, and question scheduling respects local hours

## Tech stack

- **Python 3.9+**
- **Bot** — `python-telegram-bot` v20+
- **Database** — SQLAlchemy ORM; SQLite for dev, PostgreSQL for production
- **LLM** — OpenAI (default) or Anthropic Claude (fallback)
- **Dashboard** — Flask + Jinja2, styled with Tailwind
- **Scheduler** — APScheduler (async)
- **Email** — Resend (recommended, HTTPS) or any SMTP provider
- **Hosting** — designed for Railway; also works on Fly.io, Render, or a VPS

## Quick start

### 1. Clone and install

```bash
git clone https://github.com/YOUR_USERNAME/sales-coach-bot.git
cd sales-coach-bot

python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

### 2. Configure

```bash
cp .env.example .env
# Edit .env — see docs/CONFIGURATION.md for every variable
```

Minimum required for a first run:

```
TELEGRAM_BOT_TOKEN=<from @BotFather on Telegram>
OPENAI_API_KEY=<from platform.openai.com>
ADMIN_PASSWORD=<pick something strong>
SECRET_KEY=<any random string>
APP_NAME="Your Company Coach"
COMPANY_NAME="Your Company"
```

### 3. Initialize the database and load example questions

```bash
python scripts/init_db.py
python scripts/load_questions.py data/question_banks/saas_example.json
python scripts/create_admin.py "Your Name" you@example.com <your_telegram_chat_id>
```

To find your Telegram chat_id, message your bot with `/start` after step 4 and check the logs.

### 4. Run

Two processes (in separate terminals, or use `foreman start`):

```bash
# Terminal A — Telegram bot + scheduler
python bot.py

# Terminal B — Web dashboard (http://localhost:5000)
python dashboard.py
```

### 5. Test

- On Telegram: send `/start` to your bot, then `/preguntas` to receive your first question
- On dashboard: log in with `ADMIN_PASSWORD`, browse `/reportes` to see performance data

## Adapting to your company — the 3 things to customize

**1. Question banks** (`data/question_banks/*.json`)

The three included examples (SaaS, ecommerce, professional services) are starting points. Write your own JSON file with your product questions and load it with `scripts/load_questions.py`. See `docs/QUESTION_BANK_FORMAT.md`.

**2. Knowledge base** (`data/knowledge_bases/*.txt`)

Drop plain-text product documentation, FAQs, or playbooks here. Each file's name must match the `product` tag on your questions. The LLM uses these files as ground truth when grading.

**3. Product/service tags** (`.env`)

Set `VALID_SERVICES=your,product,tags,here` to declare what your reps sell. These become filter options in the dashboard and question forms.
